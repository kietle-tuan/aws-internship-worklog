# SHARING AND FEEDBACK

## 1. Overview

This section summarizes my sharing and feedback during the AWS Cloud Computing internship and the AWS Stock Analyzer project.

During the internship, I shared my learning progress, discussed project issues with team members, and received feedback about my testing work.
As a QA Tester, my main responsibility was to check the system behavior, record testing observations, and provide feedback to support project improvement.

The feedback process helped me understand the importance of communication in a software project, especially when working with frontend, backend, and cloud-related components.

---

## 2. Sharing During the Internship

During the internship, I shared my progress mainly through weekly worklogs and group discussions.

In the first weeks, my sharing focused on what I learned from AWS tutorials and learning materials.
I discussed basic AWS concepts such as IAM, EC2, S3, VPC, Security Groups, Lambda, API Gateway, and CloudWatch.

From Week 7, after the group selected the AWS Stock Analyzer project, my sharing became more related to testing activities.
I shared testing observations, possible issues, and suggestions with the team.

The main topics I shared included:

* Basic AWS concepts that I reviewed.
* Project testing direction.
* Frontend interface issues.
* Backend API response behavior.
* Input validation cases.
* Frontend-backend integration observations.
* Final testing results.

---

## 3. Feedback I Provided to the Team

As the QA Tester, I provided feedback based on manual testing results.

My feedback mainly focused on whether the application worked correctly from the user’s perspective.
I checked the application flow, tested different input cases, and observed how the system responded.

The feedback I provided included:

* Whether the frontend displayed correctly.
* Whether buttons and input fields worked as expected.
* Whether valid stock symbols returned expected results.
* Whether empty or invalid input cases were handled properly.
* Whether backend responses were displayed correctly on the frontend.
* Whether error messages were clear enough for users.
* Whether the main user flow was easy to understand.

This feedback helped the team review the project and improve some parts before final submission.

---

## 4. Feedback I Received

During the project, I also received feedback from team members and project discussions.

The feedback helped me understand that testing should be more organized and should not only depend on random checking.
I learned that test cases should include both normal cases and error cases.

Some feedback I received included:

* Testing should cover valid and invalid input cases.
* Testing results should be recorded clearly.
* Frontend and backend should be checked together, not separately only.
* Error cases should be considered before final review.
* Testing should focus on the main functions first before checking smaller details.

This feedback helped me improve my testing approach and organize my observations more clearly.

---

## 5. Lessons Learned from Sharing and Feedback

Through the sharing and feedback process, I learned that communication is very important in a team project.
Even when my role was mainly QA Tester, I still needed to understand the project architecture, frontend behavior, backend responses, and expected system flow.

I also learned that feedback should be clear, specific, and based on actual testing results.
Instead of only saying that something did not work, it is better to describe what was tested, what result was expected, and what actually happened.

Important lessons I learned include:

* Clear communication helps the team fix issues faster.
* Testing feedback should be based on real observations.
* A tester should understand both user behavior and system behavior.
* Recording test results makes final review easier.
* Team discussion helps improve the quality of the project.

---

## 6. Personal Reflection

The sharing and feedback process helped me become more confident in my QA Tester role.
At first, I only had a basic understanding of AWS and software testing.
However, through discussion, testing, and feedback, I gradually understood how to check the application more carefully.

I also realized that testing is not only about finding bugs.
It also helps the team confirm whether the application works as expected and whether users can interact with the system smoothly.

Although my testing experience is still at a beginner level, I completed my assigned testing responsibilities and learned how to communicate testing results more clearly.

---

## 7. Conclusion

Overall, sharing and feedback were important parts of the AWS Stock Analyzer project.
Through weekly progress updates, group discussions, and testing feedback, I was able to support the team from the QA Tester perspective.

This process helped me improve my communication, documentation, and manual testing skills.
It also helped me understand that good teamwork and clear feedback are necessary for completing a software project successfully.
